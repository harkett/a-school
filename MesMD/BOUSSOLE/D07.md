# D07 — Affinage interactif de séquence

**Statut :** 🧪 Plumbing dormant, route à câbler · non commité
**Item sommaire :** [37](../TABLEAU-DE-BORD.md#item-37)

## Objectif

Permettre au prof de **piloter** l'affinage de sa séquence par instruction libre — *« phase 3 trop courte »*, *« remplace cet exercice par un jeu de rôle »*, *« ajoute un temps de bilan à la fin »*. Avec mini-pagination V1 / V2 / V3 / Vn pour revenir en arrière.

## Ce qui existe / ce qui manque

**Existe :**
- Le Générateur de séquences (`POST /api/generate-sequence`) génère une séquence en un coup.
- L'Optimiseur de séquences (`POST /api/optimize-sequence`) propose une optimisation **automatique figée** (6 critères pédagogiques) qui remplace le résultat.

**Manque :** le prof ne peut pas **piloter** l'affinage avec son propre besoin.

## Implémentation

### Frontend
Après V1 générée :
- Bouton « Affiner » → zone de saisie (texte + dictée Whisper) **sous le résultat** (pas modale, pour garder la séquence visible)
- Submit → V2 affichée
- Mini-pagination V1 / V2 / V3 / Vn pour revenir en arrière
- Tout en **state React, perdu au refresh**
- Sauvegarder écrit la version courante dans `SequenceSauvegardee` comme aujourd'hui

### Backend
Nouveau `POST /api/affiner-sequence` :
- Input : `{sequence_actuelle, instruction, matiere, niveau, theme, duree, mode}`
- Output : markdown V2
- Prompt dédié qui force *« conserve la structure générale, applique uniquement l'instruction du prof »*

## Lien avec le versioning ([D26](D26.md))

Cette mécanique « naviguer entre versions » est **éphémère** ici (state React, perdu au refresh). Le versioning ([D26](D26.md)) la **persiste** en BDD. D07 est le prérequis : sa mécanique sera réutilisée par D26.
