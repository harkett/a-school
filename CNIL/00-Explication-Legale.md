
# Explications légales — A‑SCHOOL / AFIA Services
Ce document explique clairement les obligations légales françaises (LCEN, RGPD, CNIL) et la structure recommandée pour organiser les pages légales de ton site.


# 1. Ce qui est légalement obligatoire

## 1.1 Mentions légales (LCEN)
Obligatoires pour tout site édité par une personne physique ou morale.

Doivent contenir :
- identité de l’éditeur ;
- hébergeur ;
- contact ;
- propriété intellectuelle ;
- responsabilités.

## 1.2 Politique de confidentialité (RGPD)
Obligatoire dès qu’il y a collecte de données personnelles.

Doit expliquer :
- quelles données sont collectées ;
- pourquoi ;
- combien de temps ;
- droits des utilisateurs ;
- sécurité ;
- contact pour exercer les droits.

## 1.3 Cookies
Obligatoire **uniquement** si le site utilise des cookies non techniques.

Dans le cas d’A‑SCHOOL :
- uniquement cookies techniques → **pas de bandeau** ;
- mais une **page Cookies** est recommandée pour transparence.

---

# 2. Ce qui est facultatif mais recommandé

## 2.1 CGU (Conditions Générales d’Utilisation)
Pas obligatoires légalement, mais fortement recommandées pour :
- encadrer l’usage ;
- définir les responsabilités ;
- protéger AFIA Services en cas d’abus.

## 2.2 Page Contact
Recommandée par la CNIL pour :
- demandes générales ;
- demandes RGPD ;
- support technique.

---

# 3. Faut-il un document séparé par rubrique ?
## Réponse : **NON, ce n’est pas obligatoire.**

La loi impose **l’information**, pas **le nombre de pages**.

### Tu peux donc :
- regrouper tout dans un seul fichier → conforme  
- séparer en plusieurs pages → recommandé pour clarté  

Les deux options sont légales.

---

# 4. Recommandation CNIL : structure claire et accessible

La CNIL recommande :
- une structure simple ;
- des pages accessibles en 1 clic depuis le footer ;
- une séparation claire des informations.

---

# 5. Plan de site légal recommandé (CNIL)

Voici la structure recommandée pour ton site A‑SCHOOL :
## Rôle de chaque page :

- **/mentions-legales** → obligations LCEN  
- **/politique-de-confidentialite** → obligations RGPD  
- **/conditions-generales-utilisation** → cadre contractuel  
- **/cookies** → transparence sur les cookies techniques  
- **/contact** → point d’entrée pour les droits RGPD et le support
