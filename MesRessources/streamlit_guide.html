📘 Guide Streamlit — De zéro à une vraie app

Version couverte : Streamlit 1.56.0 (avril 2026)
Public : débutant Python avec expérience en programmation

1. C'est quoi Streamlit ?

Streamlit est une bibliothèque Python open source qui transforme un script Python ordinaire en application web interactive, sans écrire une seule ligne de HTML, CSS ou JavaScript.

Tu écris du Python → Streamlit génère l'interface.

Cas d'usage typiques :

  • Tableaux de bord de données
  • Démos de modèles de machine learning
  • Outils internes (rapports, explorations de données)
  • Prototypes d'applications IA

Philosophie : "Un script = une app."
À chaque modification du script, l'app se recharge automatiquement dans le navigateur.

2. Installation

pip install streamlit

Vérifier l'installation :

streamlit hello

Cela ouvre une app de démonstration dans le navigateur. Si ça s'affiche, c'est bon.

Prérequis : Python 3.10 ou supérieur (depuis la v1.56.0).

3. Lancer une app

Crée un fichier app.py, puis lance :

streamlit run app.py

L'app s'ouvre à l'adresse http://localhost:8501.
Streamlit surveille le fichier : chaque sauvegarde recharge l'app.

4. Les bases — afficher du contenu

Texte

import streamlit as st

st.title("Mon titre principal")
st.header("Un sous-titre")
st.subheader("Un sous-sous-titre")
st.text("Du texte brut, sans mise en forme.")
st.markdown("Du texte avec **gras**, *italique*, et [liens](https://streamlit.io)")
st.write("st.write() accepte à peu près n'importe quoi : texte, données, graphiques...")

Données

import pandas as pd

df = pd.DataFrame({"Nom": ["Alice", "Bob"], "Age": [30, 25]})

st.dataframe(df)        # Tableau interactif (tri, scroll)
st.table(df)            # Tableau statique
st.metric("Score", 42, delta=+5)   # Métrique avec variation

Médias

st.image("photo.jpg", caption="Une image")
st.audio("son.mp3")
st.video("video.mp4")

5. Les widgets — interactivité

Les widgets captent les actions de l'utilisateur et retournent une valeur Python.

# Bouton
if st.button("Clique ici"):
    st.write("Bouton cliqué !")

# Zone de texte
nom = st.text_input("Ton prénom :")
st.write(f"Bonjour, {nom}")

# Curseur
age = st.slider("Ton âge", min_value=0, max_value=100, value=25)

# Menu déroulant
couleur = st.selectbox("Couleur préférée", ["Rouge", "Vert", "Bleu"])

# Cases à cocher
actif = st.checkbox("Activer l'option")

# Boutons radio
choix = st.radio("Choisis", ["Option A", "Option B"])

# Sélection multiple
tags = st.multiselect("Tags", ["Python", "Data", "IA", "Web"])

# Téléversement de fichier
fichier = st.file_uploader("Importe un CSV")

# Saisie de date
from datetime import date
jour = st.date_input("Date", value=date.today())

# Saisie date + heure (nouveau en v1.52.0)
moment = st.datetime_input("Date et heure")

Principe fondamental : Streamlit réexécute tout le script de haut en bas à chaque interaction. Les widgets retournent leur valeur courante à chaque passage.

6. Mise en page

Colonnes

col1, col2, col3 = st.columns(3)

with col1:
    st.write("Colonne 1")
with col2:
    st.write("Colonne 2")
with col3:
    st.write("Colonne 3")

Sidebar (barre latérale)

with st.sidebar:
    st.title("Menu")
    page = st.selectbox("Navigation", ["Accueil", "Données", "Graphiques"])

Expander (section repliable)

with st.expander("Voir les détails", on_change=ma_fonction):
    st.write("Contenu caché au départ.")

💡 Nouveau en v1.55.0 : on_change sur les expanders, tabs et popovers permet de déclencher du code quand l'utilisateur ouvre/ferme le panneau.

Tabs (onglets)

tab1, tab2 = st.tabs(["Résultats", "Paramètres"])

with tab1:
    st.write("Contenu de l'onglet 1")
with tab2:
    st.write("Contenu de l'onglet 2")

Popover

with st.popover("ℹ️ Aide"):
    st.write("Explication contextuelle qui s'affiche en popup.")

7. Graphiques

Streamlit s'intègre nativement avec les grandes bibliothèques de visualisation.

Graphiques natifs (simples)

import pandas as pd
import numpy as np

df = pd.DataFrame(np.random.randn(50, 2), columns=["x", "y"])

st.line_chart(df)
st.bar_chart(df)
st.area_chart(df)
st.scatter_chart(df)

Matplotlib / Seaborn

import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.plot([1, 2, 3], [4, 5, 6])
st.pyplot(fig)

Plotly (interactif)

import plotly.express as px

fig = px.scatter(df, x="x", y="y", title="Mon nuage de points")
st.plotly_chart(fig)

Altair

import altair as alt

chart = alt.Chart(df).mark_circle().encode(x="x", y="y")
st.altair_chart(chart, use_container_width=True)

8. Formulaires

Un formulaire regroupe plusieurs widgets et n'envoie les valeurs qu'au clic du bouton de soumission.

with st.form("mon_formulaire"):
    nom = st.text_input("Nom")
    age = st.slider("Âge", 0, 100)
    soumettre = st.form_submit_button("Envoyer")

if soumettre:
    st.write(f"Reçu : {nom}, {age} ans")

9. Le cache — optimiser les performances

Par défaut, Streamlit réexécute tout le script à chaque interaction. Pour éviter de recharger des données lourdes à chaque fois, on utilise le cache.

@st.cache_data
def charger_donnees():
    # Exécuté une seule fois, résultat mis en cache
    df = pd.read_csv("gros_fichier.csv")
    return df

df = charger_donnees()

@st.cache_resource
def charger_modele():
    # Pour les ressources partagées (modèles ML, connexions BD...)
    import pickle
    with open("modele.pkl", "rb") as f:
        return pickle.load(f)

Règle simple :

  • @st.cache_data → pour les données (DataFrames, listes, dicts)
  • @st.cache_resource → pour les ressources lourdes partagées (modèles, connexions)

10. Le Session State — mémoriser des valeurs

Par défaut, les variables Python sont réinitialisées à chaque réexécution. Pour conserver une valeur entre deux interactions, on utilise st.session_state.

# Initialiser
if "compteur" not in st.session_state:
    st.session_state.compteur = 0

# Modifier
if st.button("Incrémenter"):
    st.session_state.compteur += 1

st.write(f"Compteur : {st.session_state.compteur}")

C'est essentiel pour : les conversations (chatbots), les workflows multi-étapes, les paniers, etc.

11. Applications multi-pages

Depuis la v1.36.0+, Streamlit gère nativement les apps multi-pages.

Structure recommandée :

mon_app/
├── app.py          ← point d'entrée principal
└── pages/
    ├── 1_accueil.py
    ├── 2_donnees.py
    └── 3_graphiques.py

Streamlit crée automatiquement un menu de navigation dans la sidebar.

Navigation programmatique avec st.navigation (depuis v1.36.0) :

# app.py
import streamlit as st

pages = [
    st.Page("pages/accueil.py", title="Accueil"),
    st.Page("pages/donnees.py", title="Données"),
]

pg = st.navigation(pages, position="top")  # position="top" : nav en haut
pg.run()

💡 Nouveau en v1.55.0 : st.Page a un paramètre visibility pour cacher des pages du menu tout en les gardant accessibles par URL.

12. Authentification (v1.42.0+)

import streamlit as st

st.login()   # Redirige vers un fournisseur OpenID Connect (Google, GitHub, etc.)

if st.user.is_logged_in:
    st.write(f"Bienvenue, {st.user.name} !")
    if st.button("Déconnexion"):
        st.logout()

La configuration du fournisseur se fait dans .streamlit/secrets.toml.

13. Chat et IA

Streamlit a des composants dédiés aux interfaces de type chatbot.

import streamlit as st

st.title("Mon Chatbot")

# Afficher l'historique
for message in st.session_state.get("messages", []):
    with st.chat_message(message["role"]):
        st.write(message["content"])

# Saisie utilisateur
if prompt := st.chat_input("Écris ton message..."):
    # Sauvegarder et afficher le message utilisateur
    st.session_state.messages = st.session_state.get("messages", [])
    st.session_state.messages.append({"role": "user", "content": prompt})

    with st.chat_message("user"):
        st.write(prompt)

    # Réponse de l'assistant
    with st.chat_message("assistant"):
        reponse = f"Tu as dit : {prompt}"  # Remplace par ton modèle ici
        st.write(reponse)
        st.session_state.messages.append({"role": "assistant", "content": reponse})

💡 Depuis v1.52.0 : st.chat_input accepte aussi des fichiers audio en entrée.

14. Nouveautés importantes — v1.55.0 et v1.56.0

Widget Binding (v1.55.0)

Synchronise automatiquement un widget avec les query parameters de l'URL :

# La valeur du slider est reflétée dans l'URL : ?age=30
age = st.slider("Âge", 0, 100, bind="age")

Très utile pour partager un état d'app via un lien.

Images cliquables (v1.55.0)

st.image("photo.jpg", link="https://exemple.com")

Couleurs CSS arbitraires dans le Markdown (v1.55.0)

st.markdown("Texte en :color[red]{rouge} et :background-color[yellow]{surligné}")

st.metric amélioré (v1.55.0)

st.metric(
    label="Chiffre d'affaires",
    value="120 000 €",
    delta="+15%",
    delta_description="par rapport au mois dernier"  # nouveau
)

Sélection multiple en un clic (v1.55.0)

st.multiselect affiche désormais un bouton "Tout sélectionner".

15. Configuration et thème

Crée un fichier .streamlit/config.toml à la racine du projet :

[theme]
primaryColor = "#F63366"
backgroundColor = "#FFFFFF"
secondaryBackgroundColor = "#F0F2F6"
textColor = "#262730"
font = "sans serif"

[server]
port = 8501
headless = true   # utile pour le déploiement

16. Déploiement

Streamlit Community Cloud (gratuit)

  1. Pousse ton code sur GitHub
  2. Va sur share.streamlit.io
  3. Connecte ton dépôt → l'app est en ligne en 2 minutes

Docker

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 8501
CMD ["streamlit", "run", "app.py", "--server.headless=true"]

Autres options

  • Snowflake (intégration native depuis 2024)
  • Heroku, Render, Railway (via Docker ou buildpack Python)
  • AWS / GCP / Azure (via conteneur)

Tester avec AppTest (framework de test intégré) Ne jamais tester
Garder les secrets dans secrets.toml Hardcoder des mots de passe dans le code
